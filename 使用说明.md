# 司法鉴定能力挑战系统 - 云端版

基于 v1.8 + Supabase 云端同步的版本。裁判和选手在不同设备也能实时互动。

## ⚠️ 重要：安全信息泄露提醒

本项目代码会公开托管在 GitHub。**绝不能**把以下信息硬编码进代码：
- Supabase Anon Key（设计上是公开的，可以放前端代码）
- GitHub Personal Token（绝对不能放！必须撤销！）
- 数据库密码
- 任何私密 API Key

部署前检查清单：
- [ ] `lib/supabase-config.js` 中只有 URL 和 anon key（OK）
- [ ] 没有 Personal Access Token
- [ ] 没有用户密码
- [ ] 没有邮箱地址

## 🚀 部署步骤

### 1. 注册 GitHub + 创建仓库
- 注册 https://github.com
- New repository: `quiz-system-cloud`
- Public（公开，因为 GitHub Pages 公开仓库才能用）

### 2. 推送代码到 GitHub

在本目录 `C:\工具\Opencode\通用\考试系统-cloud\` 打开 PowerShell：

```powershell

git branch -M main
```

（推送时密码用 Personal Access Token，不是 GitHub 密码）

### 3. 开启 GitHub Pages
- 仓库 → Settings → Pages
- Source: Deploy from a branch
- Branch: main / (root)
- Save
- 等待 1-2 分钟，访问 `https://你的用户名.github.io/quiz-system-cloud/`

### 4. Supabase 项目设置
- 注册 https://supabase.com
- 创建项目 quiz-system
- 在 SQL Editor 执行 `supabase-init.sql`（见下）
- 获取 Project URL + anon key → 填入 `lib/supabase-config.js`

### 5. Supabase 建表 SQL

在 Supabase SQL Editor 执行：

```sql
create table public.banks (
  id uuid primary key default gen_random_uuid(),
  owner_id text not null,
  name text not null,
  encrypted_data jsonb not null,
  created_at timestamp default now()
);

create table public.matches (
  id uuid primary key default gen_random_uuid(),
  bank_id uuid references public.banks(id) on delete set null,
  judge_id text not null,
  room_id text unique not null,
  total_minutes int not null,
  status text default 'pending',
  started_at timestamp,
  expires_at timestamp,
  finished_at timestamp,
  created_at timestamp default now()
);

create table public.invite_codes (
  id uuid primary key default gen_random_uuid(),
  match_id uuid references public.matches(id) on delete cascade,
  code text not null,
  used boolean default false,
  player_name text,
  used_at timestamp
);

create table public.player_progress (
  match_id uuid references public.matches(id) on delete cascade,
  player_name text not null,
  invite_code text not null,
  unlocked_modules jsonb default '[]'::jsonb,
  unlocked_questions jsonb default '[]'::jsonb,
  started_at timestamp,
  expires_at timestamp,
  finished_at timestamp,
  updated_at timestamp default now(),
  primary key (match_id, player_name, invite_code)
);

create table public.answers (
  id uuid primary key default gen_random_uuid(),
  match_id uuid references public.matches(id) on delete cascade,
  player_name text not null,
  invite_code text not null,
  question_id text not null,
  state text default 'draft',
  value jsonb,
  is_correct boolean,
  submitted_at timestamp,
  updated_at timestamp default now(),
  unique (match_id, player_name, invite_code, question_id)
);

create table public.match_results (
  match_id uuid references public.matches(id) on delete cascade,
  player_name text not null,
  invite_code text not null,
  total_score int,
  correct_count int,
  wrong_count int,
  total_questions int,
  duration int,
  finished_at timestamp,
  primary key (match_id, player_name, invite_code)
);

alter publication supabase_realtime add table public.answers;
alter publication supabase_realtime add table public.player_progress;
alter publication supabase_realtime add table public.match_results;

create index idx_matches_room on public.matches(room_id);
create index idx_invite_match on public.invite_codes(match_id);
create index idx_invite_code on public.invite_codes(code);
create index idx_answers_player on public.answers(match_id, player_name, invite_code);
```

### 6. 关闭 RLS（简化阶段）
- Supabase → Authentication → Policies
- 对每张表点击 → Disable RLS → Confirm

## 🎯 使用流程

### 裁判端
1. 打开 `https://你的用户名.github.io/quiz-system-cloud/`
2. 首页选"裁判入口"
3. **首次**：设置裁判密码（至少 4 位）
4. 进入控制台
5. **第一步：加载题库**（Excel / 加密 / 示例）
6. **第二步：设置时长**（默认 120 分钟）
7. **第三步：生成比赛**
   - 设置邀请码数量
   - **设置题库密码**（选手访问时要输入）
   - 系统自动创建比赛并上传加密题库到云端
   - 显示房间号 + 邀请码
8. **第四步：开始比赛**（启动计时）
9. **第五步**：实时查看所有选手进度
10. **第六步**：比赛结束导出 CSV

### 选手端
1. 打开同一个 URL
2. 首页选"选手入口"
3. 看到云端已连接提示
4. 输入：姓名、房间号、邀请码、**题库密码**
5. 进入答题界面（题库已自动加载并解密）

## 🔐 安全说明

### 数据流
```
裁判端:
  1. 上传题库 → Crypto 加密 → Supabase Storage/Postgres
  2. 创建比赛 → invites、matches 表

选手端:
  1. 输入邀请码 → Cloud.listInviteCodes 验证
  2. 输入题库密码 → Crypto.decrypt 加载题库
  3. 答题 → Cloud.upsertAnswer 实时上传
  4. Realtime 订阅 → 裁判端实时看到
```

### 关键点
- **题库加密**：上传时用密码加密，存储到 Supabase
- **密码验证**：客户端用 Web Crypto API 解密，无需服务端验证
- **邀请码一次性**：用过后自动标记 `used=true`
- **Replay 攻击防护**：通过 match_id + player_name + invite_code 三元组唯一标识

### 已知限制
- RLS 关闭期间，**任何人都能读写所有数据**（演示阶段可接受）
- 生产环境应该启用 RLS 并做精细权限

## 🆘 故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 首页状态显示"云端连接失败" | Supabase 配置错 | 检查 `lib/supabase-config.js` |
| 选手登录提示"该房间号不存在" | 比赛未在云端创建 | 确认裁判已生成比赛 |
| 选手登录提示"题库密码错误" | 密码输错 | 重新向裁判索取 |
| 裁判看不到选手答题进度 | Realtime 未订阅 | 检查 Supabase Realtime 是否启用 |
| GitHub Pages 部署后 404 | 路径不对 | Settings → Pages 确认 main 分支 |
| Supabase 查询 401 | anon key 错 | 重新复制 URL 和 key |
